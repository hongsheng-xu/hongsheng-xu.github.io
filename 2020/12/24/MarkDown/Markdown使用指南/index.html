标题

使用技巧: “#” + 空格 + 文本

###### 六级标题
### 三级标题
## 二级标题
# 一级标题

效果如下:

六级标题

三级标题

二级标题

一级标题

title

使用技巧: title下方 + ‘’ = “ 或 “ - “

Second Title(-的数量随意)
------

First Title(=的数量随意)
===

效果如下:

Second Title(-的数量随意)

First Title(=的数量随意)

字体样式

  1. 加粗使用技巧: “**“ +文本+ “**“

    **这是加粗文本**

    效果如下:

    这是加粗文本

  2. 斜体使用技巧: “*“+文本+”*“

    *这是斜体*

    效果如下:

    这是斜体

  3. 粗斜体使用技巧: “***“+ 文本+”***“

    ***这是粗斜体***

    效果如下:

    这是粗斜体

分割线

分割线使用技巧: 文本下方 + 3个”_”

下划线在文本下方(_3个)
___

效果如下:

下划线在文本下方(_3个)

删除线

删除线使用技巧: “~~“ + 文本 + ‘’~~“

~~我不会markdown~~

效果如下:

我不会markdown

代码插入

小块代码使用技巧: “`” + 代码语句 + “ ` “

`npm install hexo-cli --save`

效果如下:

npm install hexo-cli --save

区域代码块使用技巧: “ ```语言类型 “+ {代码块} +”```”

​```java
public class demo {
    //代码块
}
​```

效果如下:

public class demo {
    //代码块
}

tips: 常用语言类型有c、c++、java、javascript、html、css、nodejs、bash、cmd、markdown等

对齐

使用技巧: “>”+空格

> 第一列
> > 第二列
> > > 第三列

效果如下:

第一列

第二列

第三列

链接

使用技巧: “[“+文本+”]”+”(“+地址+”)”

[百度一下,点击前往百度](https://baidu.com)

效果如下:

百度一下,点击前往百度

tips: 中括号中的内容是显示的链接文字,地址可以是网路地址也可以是本机文件的绝对地址

图像

使用技巧: “!”+”[“+文本+”]”+”(“+地址+”)”

![profile](https://avatars0.githubusercontent.com/u/61302414?s=120&v=4)

效果如下:

tips: 中括号中的文本是对图像的一段描述,如果图像加载失败将会显示中括号中的文本,地址可以是网络地址也可以是本机图像文件的绝对地址

待解释

使用技巧: “[“+”^”+文本+”]”

GitHub[^ex]
[^ex]: Build software better, together

效果如下:

GitHub[^ex]

[^ex]: Build software better, together

tips: 一般待解释需要放到文档或网页的页脚进行注释

列表

  1. 无序列表使用技巧: “+” 或 “-“ + 文本

    + 无序列表1
+ 无序列表2
- 无序列表3
- 无序列表4

    效果如下:

    • 无序列表1
    • 无序列表2
    • 无序列表3
    • 无序列表4

  2. 有序列表使用技巧: 数字 + “.” + 文本

    1. 有序列表1
2. 有序列表2

    **效果如下: **

    1. 有序列表1
    2. 有序列表2

| TITLE | COL  | COL  |

<a href="https://baidu.com/">百度一下</a>

百度一下

转义

当你想使用一些特殊字符时如: > < #等,可能会被markdown语法解析下面有两种方法防止语法被解析:

  1. 使用``反引号`来防止被解析,反引号位于esc键下方,被两个反引号包裹的内容不会被解析

    `> < # ![图片](src) [链接](href)`

    效果如下:

    > < # ![图片](src) [链接](href)

  2. 使用转义字符: \来防止语法被解析,如下:

    \> 
\<
\\
\#

    效果如下:

    >
    <
    \
    #

    tips: 如果想要使用反引号可以使用&#96;来表示 `
