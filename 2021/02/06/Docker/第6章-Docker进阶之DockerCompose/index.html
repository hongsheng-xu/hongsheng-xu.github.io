Docker Compose

  • 轻松高效管理容器-定义和运行多个容器

  • compose是开源项目需要先安装

  • 三个步骤

    1. Dockerfile定义应用环境–保证应用可以在任意地方可运行

    2. 将配置写在docker-compose.yml文件中,类似如下:

      version: "3.9"  # optional since v1.27.0
services:
  web:
    build: .
    ports:
      - "5000:5000"
    volumes:
      - .:/code
      - logvolume01:/var/log
    links:
      - redis
  redis:
    image: redis
volumes:
  logvolume01: {}

    3. docker-compose up运行项目即可

Compose安装

  • 官网地址: https://docs.docker.com/compose/install/

  1. 下载

    sudo curl -L "https://github.com/docker/compose/releases/download/1.28.2/docker-compose-$(uname -s)-$(uname -m)" -o /usr/local/bin/docker-compose

#推荐使用下面这个地址
curl -L https://get.daocloud.io/docker/compose/releases/download/1.25.5/docker-compose-`uname -s`-`uname -m` > /usr/local/bin/docker-compose

#如果第一步没有下载下来,我们就手动去网站下载即可!
#地址: https ://github.com/ docker/compose/releases
#下载:docker-compose-Linux-x86_64

然后将文件上传到/usr/loca1/bin/文件夹下，然后将其重命名为docker-compose


2. 授权

```shell
sudo chmod +x /usr/local/bin/docker-compose

    docker-compose version

Getting Started

  • 官网地址: https://docs.docker.com/compose/gettingstarted/
  • 流程:
    1. 创建网络,生成一个默认网络: 文件夹_default,且服务都是在同一网段下,可以通过域名互相访问
    2. 执行Dockerfile
    3. 启动服务
  • 默认服务名: 文件名_服务名_副本数量
  • 集群的服务一遍有多个副本(运行实例),项目一定是弹性的,多个副本保证高可用高并发

停止

  • 运行时停止

    Ctrl+C

  • 进入docker-compose.yml所在文件夹,通过命令停止

    docker-compose down

docker-compose.yml的编写规则

  • 官方示例: https://docs.docker.com/compose/compose-file/compose-file-v3/

    # 3层

version: '' # 版本号
services: # 服务
    服务1: # docker配置
        image:
        build:
        network:
        ...
    服务2: # 配置
    ...
# 其他配置:网络,卷挂载,全局规则
volumes:
networks:
configs:
    • 当你的服务依赖于其他应用时需要先启动其他服务使用depends_on

一键部署WordPress

  1. 新建一个WordPress的文件夹

    mkdir MyWordPress

  2. 新建docker-compose.yml

    vim docker-compose.yml

    复制以下配置到yml中并按下esc后在输入:wq退出

    version: '3.3'

services:
   db:
     image: mysql:5.7
     volumes:
       - db_data:/var/lib/mysql
     restart: always
     environment:
       MYSQL_ROOT_PASSWORD: somewordpress
       MYSQL_DATABASE: wordpress
       MYSQL_USER: wordpress
       MYSQL_PASSWORD: wordpress

   wordpress:
     depends_on:
       - db
     image: wordpress:latest
     ports:
       - "8000:80"
     restart: always
     environment:
       WORDPRESS_DB_HOST: db:3306
       WORDPRESS_DB_USER: wordpress
       WORDPRESS_DB_PASSWORD: wordpress
       WORDPRESS_DB_NAME: wordpress
volumes:
    db_data: {}

  3. 运行WordPress

    # 前台运行
docker-compose up
#后台运行
docker-compose up -d
#重新打包运行
docker-compose up --build

  4. 查看WordPress站点

    你可以尝试访问服务器的公网ip: http://公网ip:8000,并开始使用WordPress
