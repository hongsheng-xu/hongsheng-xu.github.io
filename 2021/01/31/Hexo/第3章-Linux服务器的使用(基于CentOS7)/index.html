XShell登录

  1. 新建远程连接,点击右上角新建按钮

  2. 在弹出的新建窗口中,输入自已服务器的名称(可以是任意名称),然后选择协议,一般云服务器的协议是SSH,公网IP可以去阿里云找到服务器的实例查看这里以阿里云轻量服务器为例,填入服务器ip后可以点击连接,如果你没有服务器,可以在阿里云或者腾讯云,百度云选购一台,一般来说学生服务器大概在100~200yuan一年:

  3. 连接服务器,先修改服务器密码

    1. 回到XShell链接服务器,选择接受来自服务器的ssh秘钥

  • 清屏

    clear

  • 关机

sync # 将数据由内存同步到硬盘中。
shutdown #关机指令，你可以man shutdown来看一下帮助文档。例如你可以运行如下命令关机:
shutdown -h 10 #这个命令告诉大家，计算机将在10分钟后关机
shutdown -h now #立马关机
shutdown -h 20:25 #系统会在今天20:25关机
shutdown -h +10 # 十分钟后关机
shutdown -r now #系统立马重启
shutdown -r +10 # 系统十分钟后重启
reboot #就是重启，等同于shutdown -r now
halt #关闭系统，等同于shutdown -h now和 poweroff

  • 文件列表查看

    ls #查看当前目录下的文件
# -a 参数表示查看全部的文件,包含隐藏文件
# -l参数列出全部的文件包含文件属性和权限,没有隐藏文件
#Linux命令可以组合使用
ls / #查看根目录下的文件
ls -ll #列表查看当前目录下的文件
ls -al #列表查看全部文件(含隐藏文件)

    文件属性,通过ls -al列出文件的属性

    文件类型属主权限属组权限其他用户权限
    d: 目录文件r(读) w(写) x(执行)r(读) w(写) x(执行)r(读) w(写) x(执行)
    l: 链接文件rwx (可读写执行)
    -: 文件—(不可读写执行)
    b: 随机存储设备r-x(可读执行)
    c:一次性读取装置

  • 修改文件属性

    chgrp -R 属组名 文件名/文件夹名 # chgrp -R root www 将www属组改为root
chown -R 属主名 文件名/文件夹名 # chown -R root www 将www属主改为root
chmod -R xyz 文件/文件名
r:4 w:2 x:1 -:0
属主: rwx=4+2+1=7
属组: rwx=7
其他用户 ---=0
chmod 777 文件名 #将一个文件属性修改为可读可写可执行,对所有用户有效赋予最高权限

  • 文件内容查看

    cat 文件名 #从第一行开始显示
tac 文件名 #从最后一行倒着显示
nl 文件名 #显示时输出行号
more 文件名 #一页一页的显示,空格键翻页,enter向下翻一行,q键退出
less 文件名 #比more更好,能往前翻页,键盘的上下键代表翻页,enter翻一行,到文件结束按下q键退出,输入字符/字符串查找/后的字符串在文件中的所有位置(向下查询),使用?字符串向下查找,例: /set (查找此页及后的所有set) ?set (查找此页及前的set字符) n表示查找下一个,N表示查找上一个
head -n 20 文件名 #只看头20行
tail -n 20 文件名 #只看尾20行
man [命令] #man cp查看cp的使用文档
#cat /etc/passwd 账号密码信息
#cat /etc/shadow 密码信息都是经过加密的

  • 切换目录

    cd 文件夹名 #进入文件夹:cd usr
cd 绝对路径 #切换到该路径:cd /usr/local/src,其中:cd / 切换到根目录,cd ~ 切换到/root
cd 相对路径 #切到该路径下的目录: cd ./usr/
cd .. #返回上一级
cd ../#返回上一级目录
pwd # 查看当前目录

  • 删除文件

    rm 文件名 #删除文件
-f 忽略不存在的文件,不会出现警告强制删除
-r 递归删除目录
rm -rf / #删库跑路!
rm -rf 文件夹 #删除文件夹下的所有文件
rmdir 文件夹 #删除为空的文件夹
rmdir -p 文件夹 #删除文件夹包含里面的内容

  • 创建文件

    mkdir 文件夹名 #创建文件夹
mkdir 文件夹1/文件夹2/文件夹3 #在当前目录下创建文件夹1,文件夹1下创建文件夹2,文件夹2下创建文件夹3

  • 复制文件

    cp 文件路径1/文件名 文件路径2 #将文件从文件路径1拷贝到文件路径2

  • 移动文件

    mv 文件夹或文件名 文件路径 #将文件夹或文件移动到另一个文件路径下
mv 文件夹1或文件名1 文件夹2或文件名2 #将文件夹或文件名更名 
-f 强制移动
-u 只替换更新的文件夹

  • 网络

    #centOS7中网络配置文件在 cd/etc/sysconfig/network-scripts
ifconfig #查看网络配置
ping #ping一个网址

  • 主机

    hostname #查看主句名称
hostname 名称 #修改名称,重启后生效

  • 修改环境变量

    vi /etc/profile #修改全局环境变量
:w #保存修改的环境变量
:q #退出
:wq # 保存环境变量并退出

  • 硬链接和软链接

    ln 文件路径1 文件路径2 #创建软链接(符号链接)--删除源文件链接失效
ln -s 文件路径1 文件路径2 #创建硬链接--删除源文件链接依然有效
touch 文件名 #创建文件
echo "字符串" >> 文件名 #向文件写入字符串

  • vim编辑器(简略)

    vim 文件名/路径下的文件 #如果文件不存在新建文件并编辑,如果存在则直接编辑
#按下i进入编辑模式
#按下:进入底线命令模式,如果是在编辑模式下要先按esc
#编辑命令
数字<space> #输入数字后按下空格光标移动n个位置
数字<enter> #输入数字后按下回车光标移动n行
0或者home键 #回到文件开头
$或者end键 #回到文件末尾
u #复原前一个动作
#底线命令
:w #保存修改的文件变量
:q #退出
:wq #保存文件并退出
:wq! #强制保存并退出
:set nu #设置行号
:set nonu #取消设置行号
#ctrl+z退出vim

  • 账号管理

    useradd -m 用户名 #自动创建该用户的主目录
-c "描述" #指定一段描述
-d #指定用户主目录,如果主目录不存在需要使用-m
-g #指定用户组
-G #指定用户的附加组
-m #使用者目录不在则自动创建
-s Shell文件 #指定用户登录的shell
-u #指定用户的用户号如果有-o选项可以重复使用其他用户的标识号
userdel -r 用户名 #删除用户及其目录
-r #清空目录
usermod 对应修改内容 用户名
usermod -d /home/xu honshen #将honshen用户的目录honshen修改为xu
su 用户名 #切换用户,su -用户名 切换到用户且到用户目录下
password 用户名 #通过root创建账户后修改账户密码,Linux输入密码是不会显示的
password #修改自己账户的密码
password -l 用户名 #锁定一个用户不能登录
password -u 用户名 #解锁一个用户
password -d 用户名 #清除密码,清除后不能登录

  • 用户组

    #用户组的增删改查是对 /etc/group文件的更新
groupadd 用户组名 #添加一个用户组,id从1000开始
groupadd -d  1003 用户组名 #添加一个用户组,id为1003
groupdel 用户组名 #删除一个用户组
groupmod -g id名 -n 新用户组名 用户组 #给一个用户组更名更id
$ newgrp root #登录当前用户切换到root组

  • 磁盘

    df #列出文件整体使用量(以字节为单位)
df -h #列出文件整体使用量(以M为单位)
du #检查磁盘空间使用量
du -a #检查磁盘及子文件夹
du -sm  /* #检查根目录下的文件占用的容量,一般/usr占用资源最大

  • 外部设备

    mount /dev/xu /mnt/xu #将名为xu的u盘挂载到mnt目录下,来实现访问u盘
umount -f 挂载位置#强制卸载u盘

  • 进程

    ps #查看当前系统进程
ps -help #查看命令的帮助
-a #显示所有进程
-u #i
-x #显示后台进程的参数
grep #查找文件中符合条件的字符串
# | 是管道符A|B将A的结果拿给B
ps -aux|grep nodejs #查看所有nodejs进程
ps -ef|grep mysql #查看mysql的父进程
pstree -pu
ps -ef | grep $USER #查看用户进程
ps -aux|grep 进程|awk '{print $2}' #查看进程号,如:ps -aux|grep nodejs|awk '{print $2}'
kill -9 `ps -aux|grep 进程|awk '{print $2}'` #杀掉指定进程如: kill -9 `ps -aux|grep hexo|awk '{print $2}'`
-p -u #-p显示父id -u显示用户组
kill -9 进程id #强制结束进程

  • 防火墙

    #查看firewall服务状态systemctl status firewalld
#开启、重启、关闭、firewalld.service服务
service firewalld start#开启
service firewalld restart#重启
service firewalld stop#关闭

# 查看防火墙规则
firewall-cmd --list-all#查看全部信息firewall-cmd --list-ports#只看端口信息

#开启端口
开端口命令:firewall-cmd --zone=public --add-port=80/tcp --permanent
重启防火墙:systemctl restart firewalld.service

命令含义:
--zone #作用域
--add-port= 80/tcp #添加端口，格式为:端口/通讯协议
--permanent#永久生效，没有此参数重启后失效

nohup挂载后台进程

  • 使用命令onhup command [args...] &挂载后台进程,接着使用exit断开与服务器的连接,终端会忽略收到 HUP（hangup）信号从而不关闭其所有子进程~如下:

    onhup hexo s -p 80 &
exit

  • 我们可以使用jobs -l或ps -ef查看后台进程,例如我们查看nojs进程并输出其进程Id

    ps -aux|grep nodejs|awk '{print $2}'

  • 拿到进程号以后,我们使用fg jubNumber将该进程调至前台,并可以直接终结该进程

    我们也可以使用Ctrl+z来将一个前台命令放到后台并暂停执行

    接着我们使用bg jobNumber来将一个后台暂停的命令继续

    [root@iz2zeg6qs3stnlr0awniwwz hongsheng-xu]# nohup hexo s -p 80 &
[1] 3032
[root@iz2zeg6qs3stnlr0awniwwz hongsheng-xu]# nohup: ignoring input and appending output to ‘nohup.out’

[root@iz2zeg6qs3stnlr0awniwwz hongsheng-xu]# jobs
[1]+  Running                 nohup hexo s -p 80 &
[root@iz2zeg6qs3stnlr0awniwwz hongsheng-xu]# fg 1
nohup hexo s -p 80
^Z
[1]+  Stopped                 nohup hexo s -p 80
[root@iz2zeg6qs3stnlr0awniwwz hongsheng-xu]# bg 1
[1]+ nohup hexo s -p 80 &
[root@iz2zeg6qs3stnlr0awniwwz hongsheng-xu]#

  • 如果后台进程挂载失败,使用lsof -i:端口号去查看端口的进程,使用nestat去查看其占用的端口

    losf -i:80
    netstat -ap|grep 80

  • 接着使用kill -9 进程号去终结该进程的运行
