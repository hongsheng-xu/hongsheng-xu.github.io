Java笔记–第4章面对象编程(Java核心)

面向过程&面向对象(OOP)

  • 面向过程思想
    • 步骤简单,第一步做什么,第二步做什么……
    • 面对过程适合处理一些较为简单的问题
  • 面向对象思想
    • 物理类聚,分类的思维模式,首先解决问题需要哪些分类,然后对这些分类进行单独思考,最后对分类下的细节进行面向过程的思索
    • 面向对象审核处理复杂问题,审核处理多人协作
  • 面向对象编程的本质: 以类的方式组织代码,以对象的组织(封装)数据
  • 三大特性: 封装、继承、多态
  • 类和对象的关系
    • 类是一种抽象的数据类型,它是对某一类事物整体描述/定义,但不能代表某一个具体的事物
    • 对象是抽象概念的具体实例

方法详解

  • 静态方法和非静态方法

    • 静态方法的修饰符static,非静态方法不加static
    //静态方法

    }
}
//demo2
public class Demo2{
    Demo1.add();
}
//非静态方法
//demo1
public class Demo1{
    //方法里除static修饰的方法都可以直接调用add()
    public int add(){
       //执行语句
    }
}
//demo2
public class Demo2{
    //首先实例化对象
    //对象类型 对象名 = 对象值
    Demo1 demo1 = new Demo1();
    demo1.add();
}
    public class Demo{
    /*
    a,b方法中可以互相调用对方
    public void a(){}
    public void b(){}
    */
    /*
    a不能直接调用b,但是b可以调用a
    原因: static是和类一起加载,不加static则是实例化后方法才存在
    public static void a(){}
    public void b(){}
    */
    //可以互相调用
    public static void a(){
        //也可以Demo.b()
        b();
    }
    public static void b(){
        //也可以Demo.a()
        a();
    }
}

三大特性

封装

  • 通常应禁止直接访问一个对象中主句的实际表示,而应通过操作接口来访问,这称为信息隐藏,隐藏代码的实现细节

  • 属性私有 get/set

  • 高内聚,低耦合:类的内部数据操作自己完成不允许外部干涉,仅暴露少量方法给外部

    public class Student {
    //不能通过实例操作name
    private String name;
    //可以通过实例.age操作age
    int age; //相当于public int age;
    //提供方法去操作数据,而不是直接用.去访问
    public void setName(String name){
        this.name = name;
    }
    public String getName(){
        return this.name;
    }
}
//idea中使用快捷键Alt+insert可以直接添加get和set方法继承

    继承

    • 类和类之间的一种关系,对一批类的抽象

    • 父类和子类,子类是父类的扩展,也叫派生类,用extends来表示

    • 子类可以继承父类的public方法和public属性,final不能继承

      //java中所有的类都默认直接或间接继承于object
//java只有单继承没有多继承

//子类
public class Student extends Person{
    //extends先执行父类构造器,在执行子类构造器

    }

      • super关键字
        • super调用父类构造方法必须在构造方法第一个
        • super必须只能出现在子类方法或构造方法中
        • super和this不能同时调用构造方法
        • this本身调用者,super父类对象的引用
        • this没有继承也能使用,super在继承下使用
        • this()本类方法的构造,super()父类关键字的构造
      //父类
public class Person {
    protected String name;
    public void say(){
        System.out.println("nice!");
    }
}
//子类
public class Student extends Person{
    public void sayName(){
        //关键字super,调用父类的属性但不能调用私有属性和方法
        System.out.println(super.name);
        this.say();//子类的say();
        super.say();//父类的say();
    }
    public void say(){}
}

      多态

      • 多态存在的条件:有继承关系,子类重写父类方法,父类引用指向子类

      • classCastException–父类子类转换异常

        //对象能执行那些方法看左边!
public class Application {
    public static void main(String[] args) {
        Son son=new Son();
        //子类能够调用的方法来自父类和子类本身
        son.test1(); //Son.test1()
        //父类的引用指向子类,向上转型,调用方法来自本身,但不能调用子类方法
        Father father=new Son();
        father.test1(); //Father.test1()
        //高转低,强制转换调用子类的放阿飞
        ((Son)father).test2(); //Son.test2()
    }
}
//父类
public class Father {
    public void test1(){
        System.out.println("Father.test()1");
    }
}
//子类
public class Son extends Father{
    public  void test1(){
        System.out.println("Son.test1");
    }
    public void test2(){
        System.out.println("Son.test2");
    }
}

方法的重写

  • 需要有继承关系,方法名必须相同,参数列表需要相同
  • 重写的修饰符范围可以扩大但是不能缩小
  • 重写抛出的异常范围可以缩小但是不能扩大
  • 不能重写的方法
    • static静态
    • final常量池,final类不能继承
    • private私有
//主程序
public class Application {
    public static void main(String[] args) {
        Son son=new Son();
        son.test(); //Son.test()
        //父类的引用指向子类,向上转型
        Father father=new Son();
        father.test(); //Father.test()
    }
}
//父类
public class Father {
    public void test(){
        System.out.println("Father.test()");
    }
}
//子类
public class Son extends Father{
    //Father father=new Son();
    //非静态的方法才可以被重写,子类重写了父类的方法
    @Override //override重新这是注解
    public void test() {
        super.test();
    }
    /*
    这是原本子类的方法被重写后变成了上面的方法
    public  void test(){
        System.out.println("Son.test()");
    }
    * */
}

static关键字

public class Father {
    static {
        //静态代码块,用于初始化
        //和类一起加载并执行一次,且只执行一次
    }
    {
        //匿名代码块,用于赋初始值,在每次生成实例对象时都会调用
    }
    //加了static的属性称为类变量,或静态变量
    //静态变量是所有实例共享的变量
    private static int age;
    //非静态变量
    private double height;

    public void test(){
        Father.age;//类变量可以直接调用
        Father father = new Father();
        //非静态变量需要实例化才能使用属性
        father.height;
        father.age;
    }
}

接口

  • 普通类: 只有具体实现

  • 抽象类: 具体实现和规范(抽象方法)都有

  • 接口: 只有规范

    //接口1
public interface UserServer {
    //接口中不能写方法的实现,方法都是public abstract
    void run(String name);
    //private void test();//报错
    //接口中定义均为public static final
    int age=10; //public static final int age=10

    public void run(String name) {

    }

    @Override
    public void time() {

    }

内部类

成员内部类

  • 在A类中定义一个内部类,B对A来说就是内部类

    //外部类
public class Outer {
    private int id;
    public void out(){
        System.out.println("Outer");
    }
    //内部类
    public class Inner{
        public void in(){
            System.out.println("In");
        }
        //内部类可以访问外部内的私有属性
        public int getId(){
            return id;
        }
    }
}
//Application
public class Application {
    public static void main(String[] args) {
       Outer outer = new Outer();
       //通过外部类来实例化内部类
        Outer.Inner inner = outer.new Inner();
        inner.in();//In
        System.out.println(inner.getId());//0
    }
}

异常机制

java把异常当做对象处理,定义了一个基类java.lang.Throwable作为所有类的超类

异常类分为:错误(ERROR)和异常(Exception)

异常处理常用五个关键字(try,catch,finally,throw,thorws)

  • 异常类型,catch参数

public static void main(String[] args) {
       int a=10;
       int b=0;
       try{//监控异常
           System.out.println(a/b);
       }catch(ArithmeticException e){
           //catch中的参数是想要捕获的异常类型
           //最高类型Throwable
           //捕获异常
           System.out.println(e);//java.lang.ArithmeticException: / by zero
        }finally {
           //程序无论是否出现异常都会执行的代码块
       }
}
//java支持依次捕获异常
try{
    2/0;//Exception异常
}catch(Error e){
}catch(Exception e){
    //此处将捕获到2/0异常
}catch(Throwable e){
    //按照异常类型从小到大写
}
//主动抛出异常
public class Application {
    public static void main(String[] args) {
       int a=10;
       int b=0;
       Application.divide1(a,b);
       Application.divide2(a,b);
    }
    public static int divide1(int a,int b){
        if (b==0){
            //一般在方法中主动抛出异常
            throw new ArithmeticException();
        }
        return a/b;
    }
    //使用throws在方法上抛出异常
    public  static int divide2 (int a, int b) throws ArithmeticException{
        return a/b;
    }
}

自定义异常

//自定义异常
public class MyException extends Exception{
    private int index;
    public MyException(int index) {
        this.index = index;
    }
    //打印异常信息
    public String toString(){
        return "MyException {" +
                "ArrayIndexOutOfBounds: "+index+
                "}";
    }
}
//抛出异常
public class Application {
    public static void main(String[] args) throws MyException {
        int index = 11;
        if(index>10){
            throw new MyException(index);//Exception in thread "main" MyException {ArrayIndexOutOfBounds: 11}
        }
    }
}
